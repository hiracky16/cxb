# cxb (Context Builder)

**"Stop feeding your AI stale context."**

`cxb` は、ドキュメントの「鮮度」と「品質」を管理し、人間と AI の両方に最適なコンテキストを提供する CLI ツールです。
ドキュメントを "Code" として扱い、Git のコミット履歴と Markdown のリンク構造に基づいて、コードの変更に追従できていない「腐敗したドキュメント」を自動で検知・可視化します。

## 課題

多くのプロジェクトでは、コードは頻繁に更新される一方で、ドキュメントの更新は忘れられがちです。その結果、以下のような問題が発生します。

-   **開発者:** 古いドキュメントを読んでしまい、混乱や手戻りが発生する。
-   **AI:** AI コーディングアシスタントに古いコンテキストを学習させてしまい、不正確なコードを生成する。

`cxb` は、ドキュメントとコードの依存関係を定義し、Git のタイムスタンプを使ってドキュメントの「鮮度」を機械的にテストすることで、この問題を解決します。

## 主な機能

-   `cxb init`: `cxb.yml` 設定ファイルを生成し、プロジェクトを初期化します。
-   `cxb check`: CI/CD の品質ゲートとして機能し、古いドキュメントや壊れたリンクをチェックします。
-   `cxb docs`: ドキュメントの構造と健全性（Healthy, Stale, Broken）を可視化するローカル HTML ダッシュボードを生成します。
-   `cxb build`: AI モデルにコンテキストを提供するために、ドキュメントを単一のファイルにエクスポート・バンドルします。

## 技術スタック
